## 개선 임팩트

### 문제별 영향도

```
신규 사용자 이탈 (온보딩 부재)
└─ 범위: 신규 사용자 100%
└─ 영향: 첫 인상 → 회원가입 결정
└─ 해결: 모달 튜토리얼 (1시간)

오류 메시지 불친화
└─ 범위: 모든 오류 상황
└─ 영향: 문제 해결 능력 저하
└─ 해결: 메시지 재작성 (1시간)

감정 분석 이해 부족
└─ 범위: 모든 응답 카드
└─ 영향: AI 신뢰도 평가 어려움
└─ 해결: 시각화 개선 (3시간)

통계 기능 미발견
└─ 범위: 활성 사용자 70%
└─ 영향: 재방문율 저하
└─ 해결: UI 구현 (5시간)
```

### 예상 수치 개선

| KPI | Before | After | 개선율 |
|-----|--------|-------|--------|
| 신규 사용자 이탈율 | 45% | 27% | ⬇️ 40% |
| 감정 분석 이해도 | 50% | 80% | ⬆️ 60% |
| 재방문율 (7일) | 30% | 48% | ⬆️ 60% |
| 오류 해결율 | 40% | 85% | ⬆️ 112% |

| 전체 만족도 | 7.5/10 | 9.0/10 | ⬆️ 20% |

## 우선순위별 실행 계획

### Week 1: Foundation (기초 다지기) - 5.5시간
```
Day 1: 온보딩 + 오류 메시지 (3시간)
       ✓ 첫 방문 시 모달 튜토리얼
       ✓ 오류 메시지 6가지 개선

Day 2: 시각화 개선 (2.5시간)
       ✓ 감정 계층 들여쓰기 + 화살표
       ✓ 신뢰도 배지 레이블 추가
```

**결과**: 신규 사용자 경험 50% 향상

---

### Week 2: Enhancement (기능 확장) - 7.5시간
```
Day 3-4: 통계 대시보드 (5시간)
         ✓ 탭 네비게이션
         ✓ CSS 바 차트
         ✓ 감정 분포 + 상황 분석

Day 5: 피드백 루프 (2.5시간)
       ✓ 👍/🤔 버튼
       ✓ 감정 수정 모달
```

**결과**: 사용자 참여도 +60%

---

### Week 3-4: Polish (마무리) - 6.5시간
```
Day 6: 모바일/데스크톱 최적화 (2시간)

Day 7: 접근성 + 다크모드 (4시간)

       ✓ 최종 QA
```

## 개선 전/후 비교

### 온보딩 (신규 사용자)

**Before**:
```
[진입] → [빈 입력 폼] → "이게 뭐지?" → [이탈]
```

**After**:
```
[진입] → [모달]
        제목: "오늘의 마음을 기록해보세요"
        내용: 3단계 사용 방법
        CTA: "시작하기" → [일기 작성]
```

---

### 오류 메시지

**Before**:
```
"응답을 해석하지 못했습니다. 다시 시도해주세요."
```

**After**:
```
API 타임아웃 → "AI가 응답하는 데 시간이 걸리고 있습니다. 1-2초 후 다시 시도해주세요."
글자 수 초과 → "일기는 500자 이내로 작성해주세요. 현재 {length}자입니다."
네트워크 오류 → "인터넷 연결을 확인해주세요."
```

## 팀별 영향도

### Backend Team
- **영향**: 낮음 (API 이미 완성)
- **작업**: onerror 메시지 매핑 추가 (30분)
- **신규 엔드포인트**: 없음

### Frontend Team
- **영향**: 높음 (UI/UX 대부분)
- **작업**:
  - 온보딩 모달 (1.5시간)
  - 시각화 개선 (2시간)
  - 통계 대시보드 (4시간)
  - 피드백 UI (2시간)
- **총 시간**: ~10시간

### Design Team
- **영향**: 중간 (스타일 가이드라인)
- **작업**:
  - 다크 모드 토큰 정의 (1시간)
  - 컴포넌트 라이브러리 문서화 (선택)
- **총 시간**: ~1시간

### QA Team
- **영향**: 높음 (새 기능 검증)
- **테스트 계획**:
  - 온보딩 흐름 (모바일/데스크톱)
  - 오류 시나리오 (6가지)
  - 통계 대시보드 (차트 정확도)
  - 피드백 루프 (데이터 저장)
  - 접근성 (WAVE, Axe)

## 기술 부채 해결

현재 문제점 중 기술적 개선 필요 사항:

1. **HTML 구조 분산**
   - 현재: 1,392줄 단일 파일
   - 추천: 컴포넌트 분리 (Phase 6)
   - 예: onboarding.html, stats.html

2. **CSS 인라인 유지보수성**
   - 현재: style 태그 내 1,000+ 줄
   - 추천: 변수/믹스인 체계화
   - 예: grid system, spacing utilities

3. **JavaScript 복잡도**
   - 현재: 내용이 많음 (검색, 필터, API 호출)
   - 추천: 모듈화 (utils, api, ui)
   - 예: `const api = { analyze(), entries() }`

4. **성능 최적화**
   - Google Fonts 로딩 (font-display: swap)
   - CSS 파일 분리
   - Chart.js lazy load
