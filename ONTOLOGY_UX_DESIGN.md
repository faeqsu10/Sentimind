# 온톨로지 기술 통합 AI 공감 다이어리 UX 설계서

> 작성일: 2026-03-05
>
> 목표: 온톨로지 기반 감정 인식을 백그라운드에서 실행하며, 사용자는 추가 복잡성 없이 더 정확한 감정 분석과 개인화된 공감 경험을 누린다.

---

## 1. 설계 원칙 (Design Pillars)

### 1.1 사용자는 온톨로지의 존재를 인식하지 않는다
- 백그라운드에서 조용히 작동 (Silent Magic)
- 사용자 입력 흐름은 현재와 동일하게 유지
- 성과는 보이되, 메커니즘은 숨긴다

### 1.2 신뢰와 프라이버시 우선
- 타인 데이터 노출 금지 (통계만 활용)
- 개인화는 사용자의 명시적 동의하에서만
- 감정 데이터는 암호화되어 저장

### 1.3 점진적 공개 (Progressive Disclosure)
- Phase 1 (v1.0): 기본 감정 분석 + 온톨로지 백그라운드 개선
- Phase 2 (v1.5): 피드백 루프 (사용자 동의/반대)
- Phase 3 (v2.0): 개인화 메시지 + 유사 상황 추천

---

## 2. 사용자 여정 (User Journey)

### Phase 1: 기본 일기 작성 (현재 상태 유지)

```
┌─────────────────────────────────────────────────────────┐
│  사용자: "오늘 면접을 봤는데 긴장했다"                    │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
         [백그라운드: 온톨로지 감정 분석]

         감정 텍스트 추출:
         "긴장" → 불안(Anxiety) 온톨로지에 매핑

         문맥 분석:
         "면접" + "긴장" → 성취 지향적 불안
                          (Achievement-oriented anxiety)

         개인화 레이어:
         사용자의 과거 데이터 (프라이빗)
         → 유사한 상황에서의 감정 패턴 매칭
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  AI 응답 (온톨로지가 개선한 정확성)                       │
│  ─────────────────────────────────────────              │
│  감정: 긴장 + 설렘 (복합 감정 감지)                       │
│  이모지: 😰➡️😊 (변화하는 감정 표현)                     │
│                                                         │
│  공감 메시지:                                             │
│  "새로운 기회 앞에서 느끼는 긴장감, 그 안에 숨은        │
│   설렘까지 함께 느껴집니다. 당신의 준비된 모습이 있으니  │
│   잘될 거예요."                                            │
│                                                         │
│  행동 제안:                                               │
│  "면접 후 깊게 숨을 쉬며 자신의 노력을 인정해주세요."   │
└─────────────────────────────────────────────────────────┘
```

**사용자 경험**: "어? 저 감정이 더 정확하게 느껴진다"는 미묘한 만족감.
기술을 모르지만, 말이 더 와닿음.

### Phase 2: 피드백 루프 (v1.5 - 선택적)

```
[일기 저장 후 1~3초 경과]

"이 감정 판단이 맞으셨나요?"
┌──────────────────────┬──────────────────────┐
│  👍 맞아요            │  🤔 아니에요         │
│  (기존 감정 유지)     │  (온톨로지 학습)    │
└──────────────────────┴──────────────────────┘
```

**만약 사용자가 "🤔 아니에요"를 선택:**

```
┌────────────────────────────────────────────────────────┐
│  AI 응답: "어떤 감정이 더 맞으셨을까요?"               │
│  ────────────────────────────────────────────────────  │
│  ("긴장" 대신 "기대감"이 더 맞으셨나요?)              │
│                                                        │
│  ☐ 기대감                                               │
│  ☐ 희망                                                 │
│  ☐ 설렘                                                 │
│  ☐ 다른 감정 (직접 입력)                                 │
│                                                        │
│  [제출] 버튼                                            │
└────────────────────────────────────────────────────────┘
```

**백그라운드 처리**:
```
사용자 피드백 → 온톨로지 학습 레이어 업데이트
├─ 문맥 정보 저장: "면접" + "기대감"
├─ 개인화 프로필 갱신 (사용자별 감정 패턴)
└─ 유사 상황에서 다음 번엔 더 정확한 감정 추천
```

사용자는 단순히 "이 감정이 맞지 않아"라고 반응하는 것만으로,
온톨로지가 개인화 데이터를 학습한다.

### Phase 3: 개인화 메시지 + 통찰 제공 (v2.0)

```
일기 저장 후 AI 응답 섹션 하단:

┌────────────────────────────────────────────────────────┐
│  💡 당신의 감정 패턴                                    │
│  ────────────────────────────────────────────────────  │
│  지난 30일 간 당신은 "불안"을 느낄 때                   │
│  대부분 "설렘" 또는 "기대감"이 함께 나타났어요.         │
│                                                        │
│  혹시 당신은 긍정적인 변화 앞에서                       │
│  자연스럽게 기대감을 갖는 성향이 있나요?               │
│                                                        │
│  [더 알아보기] → 통찰 페이지로 이동                     │
└────────────────────────────────────────────────────────┘
```

**선택적 "영감" 섹션** (프라이버시 보호하며):

```
┌────────────────────────────────────────────────────────┐
│  ✨ 유사한 상황, 다른 감정들                             │
│  ────────────────────────────────────────────────────  │
│  "면접"과 관련된 다른 사람들의 일기들:                  │
│                                                        │
│  📌 "면접 합격 연락을 받았다" → 기쁨 (23명)            │
│  📌 "면접 준비하다 힘들었다" → 끈기 (15명)             │
│  📌 "면접 후 뿌듯했다" → 자부심 (10명)                 │
│                                                        │
│  [각 감정 일기 보기] (익명, 통계만 공개)               │
└────────────────────────────────────────────────────────┘
```

**개인화 조언** (사용자 패턴 기반):

```
일기 저장 후 "행동 제안" 섹션 확장:

기본 제안: "면접 후 깊게 숨을 쉬며 자신을 인정해주세요."

+ 당신을 위한 맞춤 제안:
  "당신은 준비된 상황에서 힘을 내는 분 같아요.
   다음 번 도전 앞에서는 '나는 잘 준비했다'는
   자신감을 먼저 느껴보세요."
```

## 4. 온톨로지 엔진 (기술 구현 로드맵)

### 4.1 감정 온톨로지 구조

```
감정 온톨로지 (Emotion Ontology)
│
├─ 기본 감정 (Primary Emotions)
│  ├─ 기쁨 (Joy)
│  │  ├─ 만족감, 행복, 뿌듯함
│  │  └─ 설렘, 희망, 감동
│  ├─ 슬픔 (Sadness)
│  ├─ 분노 (Anger)
│  ├─ 두려움 (Fear)
│  │  └─ 불안, 긴장, 걱정
│  ├─ 놀라움 (Surprise)
│  └─ 혐오 (Disgust)
│
├─ 복합 감정 (Complex Emotions)
│  ├─ 질투 = 분노 + 두려움
│  ├─ 향수 = 슬픔 + 기쁨
│  ├─ 희망 = 기쁨 + 두려움
│  └─ 공감 = 슬픔 + 기쁨
│
└─ 상황-감정 매핑 (Context Mapping)
   ├─ "면접" → [불안, 기대감, 설렘]
   ├─ "연애" → [기대감, 불안, 행복]
   ├─ "마감" → [초초함, 성취감, 피로]
   └─ "실패" → [실망, 배움, 아쉬움]
```

### 4.2 개인화 모델 (User Profile)

```
사용자 감정 프로필
│
├─ 감정 분포 (Emotion Distribution)
│  └─ 지난 30/90/365일 감정 빈도
│
├─ 감정 연관성 (Emotion Correlation)
│  └─ "불안" 다음에 "기대감"이 올 확률: 77%
│
├─ 상황별 감정 패턴 (Contextual Pattern)
│  ├─ "도전 상황": [불안 → 설렘 → 성취감]
│  └─ "손실 상황": [아쉬움 → 배움 → 감사]
│
├─ 감정 성향 (Emotional Tendency)
│  ├─ 유형: Challenge-Oriented, Harmony-Oriented 등
│  └─ 강점: 목표 달성 능력, 적응력 등
│
└─ 피드백 히스토리 (Feedback History)
   ├─ "면접+기대감" (사용자 확인)
   └─ "발표+긴장" (사용자 수정 → "자신감")
```

### 4.3 백엔드 구현 플로우

```
POST /api/analyze (온톨로지 강화 버전)

입력: { text: "오늘 면접을 봤는데 긴장했다" }
      │
      ├─ [Step 1] 기존 감정 분석 (Gemini)
      │  └─ "긴장" 추출
      │
      ├─ [Step 2] 온톨로지 매핑
      │  └─ "긴장" → Fear(공포) 온톨로지 노드
      │            → 불안(Anxiety) 세부 분류
      │
      ├─ [Step 3] 문맥 분석 (Context Enrichment)
      │  ├─ 키워드 추출: "면접"
      │  ├─ 온톨로지 컨텍스트 탐색:
      │  │  "면접" 상황 → [불안, 기대감, 설렘]
      │  └─ 예측: "기대감"도 함께 있을 가능성 높음
      │
      ├─ [Step 4] 개인화 조정 (User Profile Adjustment)
      │  ├─ 사용자 프로필 조회

      │

      │  └─ "긴장(불안) + 설렘(기대감)" → 이중 표현

      └─ 출력: {
```

결코 강압적이지 않음.

## 12. 주의사항 & 한계

### 12.1 온톨로지의 한계

```
❌ 감정은 개인마다 다름
  → 온톨로지는 "일반적 지식"만 제공
  → 개인화 학습이 필수

❌ 문화적 다양성 무시 금지
  → 한국 문화권 감정 표현 특수성 반영
  → 예: "답답함" (고유한 한국 감정)

❌ 치료 목적 금지
  → "이 앱이 의료 상담을 대체하지 않습니다"
  → 필요시 전문가 상담 안내
```

### 12.2 프라이버시 우선

```
금지 사항:
  ❌ 개인 감정 데이터 판매
  ❌ 제3자 광고주에게 노출
  ❌ 사용자 동의 없는 학습

허용 사항:
  ✅ 익명 통계 (감정 분포)
  ✅ 문맥 없는 감정 연관성
  ✅ 일반화된 통찰 (당신의 유형은...)
```

## 최종 정리

**온톨로지 기술을 통합한 AI 공감 다이어리 UX의 핵심 원칙:**

1. **조용한 혁신 (Silent Magic)**: 사용자는 기술을 모르지만 개선을 느낌
2. **점진적 공개 (Progressive Disclosure)**: 필요한 시점에 고급 기능 제공
3. **신뢰 우선 (Trust First)**: 프라이버시 > 기능
4. **개인화 학습 (Personalized Learning)**: 사용자 피드백이 AI를 성장시킴
5. **문화적 감수성 (Cultural Sensitivity)**: 한국 문화권 감정 표현 존중

이렇게 설계하면, 사용자는 "더 정확하고 따뜻한 AI"를 경험하면서도
기술의 복잡성은 전혀 느끼지 않습니다.
