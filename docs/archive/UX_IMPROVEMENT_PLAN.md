# UX 개선 계획 요약
## Sentimind: Landing + Guest Mode (Phase 5-A)

**작성일**: 2026-03-06
**상태**: 🟡 기획 단계 → 개발 대기 중

---

## 1. 3가지 핵심 문제점

### ❌ 문제 1: 로그인 강제
- **현상**: 신규 방문자가 첫 화면에서 로그인 폼 마주침
- **결과**: "이 서비스가 뭐 하는 거지?"라는 의문 → 바운스
- **해결**: Landing + Demo 모드

### ❌ 문제 2: 오프라인 알림 혼동
- **현상**: 로그인 후에도 "오프라인 상태입니다" 경고 계속 표시
- **원인**: localStorage 캐시만 써도 불필요한 네트워크 상태 UI
- **해결**: 상태 분명히 (저장 중 → 저장 완료 → 실패)

### ❌ 문제 3: 가치 미전달
- **현상**: 로그인 페이지만 있어서 서비스 매력 못 느낌
- **결과**: 회원가입 결정 어려움
- **해결**: 시각화된 Landing (스크린샷, FAQ, 통계)

## 2. 5가지 기능 (F1-F5)

### 🎯 F1. Landing Page (신규 방문자)
```
┌──────────────────────────────┐
│ Hero: 당신의 감정을 이해하는  │
│       AI 일기장             │
│                              │
│ [무료로 시작하기] [알아보기]  │
│                              │
│ 감정 분석 | 공감 응답 | 통계   │
│ (3개 기능 카드)              │
│                              │
│ 스크린샷 / FAQ / 평가        │
│                              │
│ Footer                       │
└──────────────────────────────┘
```
**목표**: 신규 방문자 3분 내 서비스 이해 + "체험하고 싶음" 유도

---

### 🎯 F2. Guest Mode (무로그인 체험)
```
[샘플 일기 3개 제시]
  ├─ 기쁨: "발표 성공!"
  ├─ 슬픔: "친구와 싸움"
  └─ 불안: "내일 면접"

[또는 직접 작성]
  ↓
[AI 감정 분석 + 공감 응답]
  ├─ 감정 분류 (3단계)
  ├─ 신뢰도
  └─ 위로 메시지

[localStorage에 임시 저장]
  └─ 최대 10개 일기
```
**목표**: "로그인 없이도 3분 내 체험 가능" → 신뢰도 UP

---

### 🎯 F3. 오프라인 알림 개선
```
[기존 문제]
  "오프라인 상태입니다" ← 게스트도, 로컬 저장도 불필요

[개선]
  게스트 모드: 알림 없음 (로컬만 사용)
  멤버 모드:
    ├─ 저장 중... (진행)
    ├─ ✅ 저장 완료 (2초 후 사라짐)
    └─ ❌ 저장 실패 (재시도 자동)
```
**목표**: 명확한 상태 표시 + 불필요한 경고 제거

---

### 🎯 F4. Guest → Member 마이그레이션
```
[게스트가 5개 일기 작성]
  ↓
[회원가입]
  ↓
[자동 마이그레이션]
  ├─ localStorage → Supabase
  ├─ 타임스탐프 유지
  ├─ user_id 설정
  └─ "5개의 일기가 저장됐습니다" ✅
```
**목표**: 데이터 손실 없이 부드러운 전환

---

### 🎯 F5. 회원가입 유도
```
[게스트: 일기 저장 시도]
  ↓
[모달: "이 일기를 저장할래요?"]
  ├─ [회원가입하기] → 계정 생성 + 마이그레이션
  ├─ [나중에] → 로컬 저장 계속
  └─ [계속 로컬로] → 게스트 유지
```
**목표**: 강제 아님, 자연스러운 전환

## 3. 우선순위 & 일정

### 우선순위
```
🔴 P0 (필수)
  ├─ F1 Landing Page
  └─ F2 Guest Mode

🟠 P1 (권장)
  ├─ F3 오프라인 알림 개선
  └─ F5 CTA 최적화

🟡 P2 (보완)
  └─ F4 마이그레이션 (고복잡도)
```

**팀**: FE 2명, BE 1명, QA 1명

## 4. 기술 제약사항 3가지

### T1. 단일 HTML 파일
- ✅ CSS/JS 인라인 추가 (기존 방식)
- ❌ 새 파일 생성 금지
- **영향**: HTML 700-1000줄 증가

### T2. Backend 최소 변경
- ✅ 신규: POST /api/entries/import (50줄)
- ✅ 기존 API 유지
- ❌ 인증 로직 변경 금지
- **영향**: 서버 코드 50줄 추가

### T3. 회귀 방지
- ✅ 로그인/회원가입 정상 동작
- ✅ 데이터 저장/조회 정상 동작
- ✅ 반응형 유지
- **테스트**: 회귀 테스트 20+ 항목

## 5. 성공 기준

### 비즈니스
- 🎯 Landing 이탈률 < 40% (70% → 30%)
- 🎯 Guest → Member 전환율 > 30%
- 🎯 평균 체험 시간 > 3분

### 기술
- ✅ 회귀 테스트 100% 통과
- ✅ 모바일 응답 시간 < 1초
- ✅ 오류 로그 < 1%

## 6. 데이터 흐름 (간단)

### Guest 모드
```
신규 방문자
  ↓ [무료로 시작하기]
Demo 화면 (authScreen 숨김)
  ↓ 샘플 일기 또는 직접 작성
AI 분석 (Gemini API)
  ↓ 응답 표시
localStorage 저장 (최대 10개)
  ↓ [회원가입] 클릭 또는 [나중에]
```

### Member 모드
```
게스트 localStorage
  ↓ 회원가입 완료 (토큰 발급)
POST /api/entries/import (마이그레이션)
  ↓
Supabase entries (user_id 설정)
  ↓
App 화면 (히스토리 표시)
```

## 8. 마이그레이션 경로 (기술)

```javascript
// 게스트 데이터
localStorage.setItem('sentimind-guest-entries', JSON.stringify([
  { id: 'guest_1', text: '...', emotion: {...}, createdAt: '...' }
]));

// 회원가입 시
await fetch('/api/entries/import', {
  method: 'POST',
  headers: { 'Authorization': 'Bearer <token>' },
  body: JSON.stringify({ entries: guestEntries })
});

// 서버: Supabase에 저장
await supabase.from('entries').insert(
  entries.map(e => ({ ...e, user_id: userId }))
);

// 결과: localStorage 정리, 히스토리에 표시
localStorage.removeItem('sentimind-guest-entries');
```

## 10. 리스크 & 완화

| 리스크 | 영향 | 해결 |
|--------|------|------|
| HTML 파일 비대화 | 유지보수 어려움 | 주석 + CSS 구조화 |
| 데이터 마이그레이션 손실 | 사용자 신뢰 하락 | 백업 + 에러 처리 |
| 오프라인 감지 실패 | 잘못된 UI | 하트비트 + 명확한 메시지 |
| 기존 로직 회귀 | 멤버 이탈 | 강화된 테스트 |
