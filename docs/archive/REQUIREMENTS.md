## 1. 현황 분석

### 문제점 3가지
1. **로그인 강제**: 신규 방문자가 서비스 가치를 느끼지 못한 채 회원가입 결정
2. **오프라인 알림**: 로그인 후에도 "오프라인 상태입니다" 알림이 계속 표시 (localStorage 로컬 캐시라 불필요)
3. **가치 미전달**: 첫 화면이 로그인 폼 → 서비스 매력 전혀 못 느낌

### 현재 구조 (분석 결과)
```
server.js
├── /api/auth/login, signup, reset-password (Supabase 기반)
├── /api/auth/me (토큰 검증)
├── /api/auth/refresh (토큰 갱신)
├── /api/entries (CRUD)
└── /api/analyze (Gemini API)

public/index.html
├── authScreen (로그인/회원가입 페이지)
├── app (메인 앱 + 통계 + 히스토리)
└── 현재: authScreen.hidden = false (기본값) → 첫 화면이 로그인
        → authScreen.hidden = true (로그인 성공 후)
```

**핵심 관찰**:
- localStorage: `sb-access-token`, `sb-refresh-token` (Supabase 인증)
- 토큰 없으면 authScreen 표시 → 게스트 모드 없음
- 오프라인 알림: 아마 네트워크 상태 감지 코드 있을 것 (온라인 상태 UI 강제)

## 3. User Stories (수용 기준 포함)

### F1. Landing Page

**User Story**:
신규 방문자로서, 사이트에 들어갔을 때 "이 서비스가 뭐 하는 거지?"라는 질문에 답변을 얻고,
3분 이내에 체험을 결정하고 싶다.

**수용 기준**:
- ✅ AC1: Hero section에 핵심 메시지 3개
  - 📌 제목: "당신의 감정을 이해하는 AI 일기장"
  - 📌 부제: "한 줄 일기 → AI 감정 분석 → 공감 응답"
  - 📌 CTA: [무료로 시작하기] 또는 [Demo 체험]

- ✅ AC2: 주요 기능 3개 카드 (아이콘 + 텍스트)
  - 🧠 감정 분석: "AI가 당신의 감정을 정확히 읽습니다"
  - 💬 공감 응답: "당신의 상황에 맞는 위로 메시지를 받습니다"
  - 📊 통계: "감정 패턴을 시각으로 추적합니다"

- ✅ AC3: 스크린샷 또는 영상 (실제 사용 흐름)
  - 📸 스크린샷 3개 (일기 작성 → AI 응답 → 통계)
  - ⏹️ 또는 GIF/영상 (15초)

- ✅ AC4: CTA 버튼 2개
  - [무료로 시작하기] → 회원가입 (또는 Demo)
  - [나중에 보기] → 페이지 하단으로 스크롤

- ✅ AC5: FAQ (5개 핵심 질문)
  - Q1: "로그인 없이 사용할 수 있나요?" → A: "네, Demo 모드로 3분 체험 가능"
  - Q2: "데이터는 안전한가요?" → A: "모든 데이터는 암호화되어 저장됩니다"
  - Q3: "비용이 드나요?" → A: "기본 기능은 무료, 프리미엄은 추후"
  - Q4: "오프라인에서도 사용 가능한가요?" → A: "네, 로컬에 저장 후 로그인 시 동기화"
  - Q5: "다른 기기에서도 일기를 볼 수 있나요?" → A: "네, 로그인 후 모든 기기에서 접근 가능"

- ✅ AC6: Social Proof (신뢰도)
  - 📊 통계: "10,000+ 사용자, 50,000+ 일기 분석"
  - ⭐ 평가: "평균 4.8/5 (구글 플레이/앱스토어 링크)"
  - 💬 후기: 사용자 평가 2-3개 카드

- ✅ AC7: Footer
  - 📄 About: "Sentimind: AI-Powered Emotion Diary"
  - 🔐 Privacy Policy (링크)
  - ⚖️ Terms of Service (링크)
  - 📧 Contact: support@sentimind.io
  - 🔗 Social: GitHub, Twitter

**설계 고려사항**:
- 모바일 최우선 (반응형 + 터치 친화적)
- 색상: 기존 UI와 일관성 (var(--color-primary) 등)
- 폰트: 기존 Gowun Batang/Dodum 사용
- 애니메이션: 부드러운 페이드/스크롤 (과하지 않음)
- SEO: title, meta description, og:image 설정

### F2. Demo/Guest 모드

**User Story**:
신규 사용자로서, 회원가입 없이 서비스를 3분 이내에 체험하고,
마음에 들면 회원가입으로 자연스럽게 전환하고 싶다.

**수용 기준**:
- ✅ AC1: 게스트 진입 경로 명확
  - [무료로 시작하기] → "Demo로 시작할래요" / "가입할래요"
  - 또는 Landing → [Demo 체험] 버튼 직접 클릭
  - 진입 후: authScreen 숨기고, guestMode 활성화

- ✅ AC2: 샘플 일기 2-3개 제시 (감정별)
  ```json
  [
    {
      "emotion": "기쁨",
      "text": "오늘 팀 프로젝트 발표 성공했다! 모두 칭찬해줘서 정말 기뻐 :)",
      "icon": "😊"
    },
    {
      "emotion": "슬픔",
      "text": "친구와 싸웠어... 내가 잘못했는데 어떻게 풀어야 할지 모르겠어",
      "icon": "😢"
    },
    {
      "emotion": "불안",
  ```

  - 새 일기 입력 가능 (localStorage에 임시 저장)
  - AI 분석 + 공감 응답 표시
  - 통계에도 반영 (임시)

  - 일기 선택/작성 후 "분석하기" 버튼

  - 응답 카드: 감정 계층 + 신뢰도 + 공감 메시지

- ✅ AC5: 회원가입 유도 (강제 아님)

  - 또는 일기 저장 클릭 시 → "이 일기를 저장할래요? (가입 필요)"
  - 버튼: [회원가입하기] / [나중에] / [계속 Demo로 사용]

- ✅ AC6: localStorage에 임시 저장
  ```javascript
  localStorage.setItem('sentimind-guest-entries', JSON.stringify([
    { id: '...', text: '...', emotion: {...}, createdAt: '...' }
  ```

  - Demo 종료 후 돌아와도 데이터 유지

- ✅ AC7: 안내 메시지 (Header 또는 Banner)
  - "👤 게스트 모드로 체험 중입니다 | 최대 10개 일기 저장 가능 | [로그인하기]"
  - 색상: 노란색/주황색 (주의 유도, 강제 아님)

**기술 구현**:
- 새 상태 변수: `isGuestMode = true/false`
- UI 분기: authScreen 대신 guestLanding + guestApp
- API 호출: 게스트 모드에선 `/api/entries` 호출 안 함 (localStorage만)
- localStorage 키: `sentimind-guest-entries`, `sentimind-guest-mode`

### F3. 오프라인 알림 개선

**User Story**:
로그인 후 불필요한 "오프라인 상태입니다" 경고를 받지 않고,
실제 저장 상태를 명확하게 알고 싶다.

**수용 기준**:
- ✅ AC1: 게스트 모드에서는 오프라인 배너 표시 안 함
  - localStorage만 사용하므로 "오프라인" 개념 불필요
  - isGuestMode === true → 네트워크 상태 UI 숨기기

- ✅ AC2: 회원 모드에서 상태 분명히 표시
  - "💾 저장 중..." (네트워크 요청 진행 중)
  - "✅ 저장 완료" (성공, 2초 후 사라짐)
  - "❌ 저장 실패 - 재시도 중" (에러 + 자동 재시도)

- ✅ AC3: 실제 저장 실패 시에만 토스트 알림
  - 네트워크 OK, 서버 OK → 아무것도 표시 안 함 (침묵은 금)
  - 에러 발생 시 → Toast: "일기 저장에 실패했습니다. (재시도 중...)"

- ✅ AC4: 자동 재시도 (지수 백오프)
  - 1차 실패 → 1초 후 재시도
  - 2차 실패 → 2초 후 재시도
  - 3차 실패 → 사용자 개입 필요 ("수동 재시도" 버튼)
  - 토스트에 표시: "재시도 1/3"

- ✅ AC5: 오프라인 상태가 오래 지속되면 명확한 안내
  - navigator.onLine === false → 배너: "📡 인터넷 연결이 끊겼습니다"
  - 5초 이상 → 자동으로 로컬 캐시 사용 ("오프라인 모드로 전환됨")
  - 재연결 시 → "📡 인터넷 복구됨, 데이터 동기화 중..."

**현재 코드 수정점**:
```javascript
// 현재: "오프라인" 배너가 있을 것 → 게스트 모드에선 숨기기
if (isGuestMode) {
  document.getElementById('offlineBanner').hidden = true;
}

// 저장 상태 UI 추가
async function saveEntry(entry) {
  showSavingUI('저장 중...');
  try {
    await fetchWithAuth('/api/entries', { method: 'POST', body: JSON.stringify(entry) });
    showSavingUI('저장 완료', 'success');
    setTimeout(() => clearSavingUI(), 2000);
  } catch (err) {
    showSavingUI('저장 실패 - 재시도 중...', 'error');
    // 지수 백오프 재시도
  }
}
```

### F4. Guest → Member 마이그레이션

**User Story**:
게스트로 작성한 일기를 회원가입 후 자동으로 클라우드에 저장하고,
이전 작업이 사라지지 않기를 원한다.

**수용 기준**:
- ✅ AC1: 회원가입 완료 후 자동 마이그레이션
  - 1) 회원가입 성공 → 토큰 발급
  - 2) localStorage의 `sentimind-guest-entries` 읽기
  - 3) POST /api/entries/import (배치 마이그레이션 엔드포인트)
  - 4) 서버에서 Supabase entries 테이블에 일괄 저장
  - 5) localStorage 정리 (sentimind-guest-entries 삭제)

- ✅ AC2: 타임스탐프 유지
  - 게스트 일기의 createdAt, updatedAt 보존
  - Supabase에 저장 시에도 동일 타임스탐프 사용
  - UI에서 "3일 전", "2주 전" 등으로 표시 (정확한 시간 유지)

- ✅ AC3: 작성자 정보 (user_id) 자동 업데이트
  - 게스트 일기: user_id = null
  - 마이그레이션 후: user_id = 현재 로그인 사용자 ID
  - 다른 사용자가 로그인해도 본인 일기만 보임 (RLS 정책)

- ✅ AC4: 성공 메시지 + 확인
  - Toast: "✅ 5개의 일기가 클라우드에 저장되었습니다"
  - 앱 화면으로 자동 이동 (히스토리 탭)
  - 리스트에 마이그레이션된 일기들 표시

- ✅ AC5: 마이그레이션 실패 시 안전한 복구
  - 서버 에러 → Toast: "❌ 일기 동기화에 실패했습니다"
  - 사용자에게 선택지 제공:
    - [수동 저장] → JSON 파일로 다운로드 (빠른 복구)
    - [다시 시도] → 마이그레이션 재실행
  - localStorage는 유지 (데이터 손실 방지)

**API 엔드포인트 (신규)**:
```javascript
// POST /api/entries/import
// Authorization: Bearer <token>
// Body: { entries: [...] }
// Response: { success: true, imported: 5, failed: 0 }
```

**서버 구현** (간단함):
```javascript
app.post('/api/entries/import', authMiddleware, async (req, res) => {
  const { entries } = req.body;
  const userId = req.user.id;

  // Supabase에 일괄 저장
  const { error } = await supabase
    .from('entries')
    .insert(entries.map(e => ({
      ...e,
      user_id: userId
    })));

  if (error) return res.status(400).json({ error: error.message });
  res.json({ success: true, imported: entries.length });
});
```

### F5. 회원가입 유도 (CTA 최적화)

**User Story**:
게스트 체험을 마친 후, "이 서비스 정말 좋은데 데이터를 계속 쓸려면?"
라는 자연스러운 질문으로 회원가입을 결정하고 싶다.

**수용 기준**:
- ✅ AC1: 자연스러운 유도 모달 (강제 아님)
  - 일기 저장 시도 → "💾 이 일기를 저장할래요?"
  - 모달: "게스트 모드에선 로컬에만 저장돼요. 클라우드에 저장하고 어디서나 접근할래요?"
  - 버튼: [회원가입하기] / [나중에] / [계속 로컬로]

- ✅ AC2: 선택지 명확
  - [회원가입하기] → 회원가입 폼 (미리 이메일 입력 있으면 자동 채우기)
  - [나중에] → 모달 닫음, 로컬 저장 계속
  - [계속 로컬로] → 게스트 모드 유지, 최대 10개까지 경고 없음

- ✅ AC3: 회원가입 폼 간단함
  - 📧 이메일 (필수)
  - 🔐 비밀번호 (필수, 8자 이상)
  - 👤 닉네임 (필수, 선택사항이 아님)
  - 동의: 이용약관 + 개인정보 (체크박스)
  - [가입하기] 버튼

- ✅ AC4: 회원가입 완료 → 자동 로그인 + 마이그레이션
  - 성공 → localStorage 정리
  - 마이그레이션 실행 (F4 참조)
  - App 화면으로 자동 이동

- ✅ AC5: 닉네임 선택 후 앱으로 스무스 이동
  - 로딩 애니메이션 (2초)

## 4. 기술 제약사항

### T1. 단일 HTML 파일 관리
- ❌ 새 파일 생성 제한 (HTML/JS/CSS 분리 불가)

- ✅ 신규 엔드포인트: POST /api/entries/import (마이그레이션)
- ✅ 기존 엔드포인트 유지: /api/auth/*, /api/entries/*, /api/analyze
- ❌ 인증 로직 변경 금지 (게스트는 토큰 없이 localStorage만)
- ❌ 데이터베이스 스키마 변경 금지 (Supabase 마이그레이션은 Phase 5 후)
- **영향**: 서버 코드 50줄 추가 (import 엔드포인트)

### T3. 기존 기능 회귀 방지
- ✅ 로그인/회원가입 정상 동작 (authScreen 로직 그대로)
- ✅ 데이터 저장/조회 정상 동작 (/api/entries)
- ✅ AI 감정 분석 정상 동작 (/api/analyze)
- ✅ 통계/히스토리/필터 정상 동작
- ✅ 반응형 디자인 유지 (모바일 + 데스크톱)
