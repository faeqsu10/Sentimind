## 로드맵 원칙

1. **사용자 가치 우선**: 기술 마이그레이션 단독 Phase는 지양. 사용자가 느낄 변화와 반드시 묶는다.
2. **조기 출시**: Phase 5+6 완료 시점(5주)에 베타 배포. 실 사용자 피드백을 최대한 빨리 수집한다.
3. **MoSCoW 우선순위**: Must 기능 완성 후 Should/Could 순서로 진행.
4. **리스크 버퍼**: 각 Phase 일정에 20-30% 버퍼 포함.

## Phase 10: AI 고급 기능 (3주)

### 목표

온톨로지 기반 감정 데이터를 활용한 개인화 인사이트와 AI 코칭 기능 추가.

### 기간

- 기간: 2026-05-07 ~ 2026-05-28 (3주)

### 주요 기능

| 기능 | 설명 | 기술 | MoSCoW |
|------|------|------|--------|
| 감정 패턴 AI 요약 | 월간 감정 리포트 (Gemini 기반 자연어 요약) | Gemini API | Should |
| 유사 일기 추천 | 과거 같은 감정의 일기 추천 | pgvector (임베딩 검색) | Could |
| 개인화 조언 | 사용자의 감정 패턴 기반 맞춤 조언 | Gemini + 감정 히스토리 | Could |
| 감정 예측 알림 | 스트레스 급증 패턴 감지 시 선제 알림 | 시계열 분석 | Could |

### 기술 요구사항

- pgvector: Supabase 확장 활성화 필요
- Embedding API: Gemini embedding-001 또는 text-embedding-004
- 저장 용량: 768차원 벡터 × 사용자당 평균 100건 = 사용자당 ~1MB

### 완료 기준

- [ ] 월간 감정 리포트 생성 (최소 30건 일기 보유 사용자 대상)
- [ ] 프리미엄 전환 사용자 대상 베타 운영
